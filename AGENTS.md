# AGENTS.md

Guidance for AI agents working in this repo. Only repo-specific, non-obvious
facts are listed; assume standard tooling otherwise.

## What this repo is

A single-source **LaTeX cheatsheet**. The only real source file is
`macos_cheatsheet.tex`; everything else is build tooling, linters, and CI.
The build output is `macos_cheatsheet.pdf` (plus JPG/PNG/SVG), published to
GitHub Releases. Build artifacts are git-ignored and never committed.

## Build

- Build with `./run.sh` (NOT a bare `make`). It runs `make` inside the Docker
  image `ghcr.io/xu-cheng/texlive-full`, so Docker is the only local
  prerequisite; no native TeX install is needed.
- Default targets: `jpg pdf png svg mostlyclean`. Build a subset with e.g.
  `./run.sh pdf` or override the image with `./run.sh -i <image> pdf`.
- `run.sh` installs `poppler-utils` in the container for image conversion.
- The image is amd64-only; CI runs LaTeX builds on `ubuntu-latest`, not arm64.

## Do not hand-edit

- `Makefile` and `.gitignore` are vendored from
  [makefile4latex](https://github.com/tueda/makefile4latex) (note the upstream
  `latest-raw-url:` tags and the `make upgrade` target). Change build behavior
  via the variables/config at the top of the `Makefile`, not the engine below.
- `CHANGELOG.md` is generated by release-please and is excluded from every
  linter. Never edit it manually.

## Editing the `.tex`

- `make lint` runs `chktex`. Suppress an unavoidable false positive with an
  inline `% chktex <NN>` comment on that line (see existing `% chktex 26`
  usages) rather than disabling the linter.
- `make pretty` runs `latexindent` to format `.tex` sources.

## Linting (MegaLinter)

CI lint is [MegaLinter](https://megalinter.io/), configured in
`.mega-linter.yml`. Tool choices that differ from defaults:

- Markdown: `rumdl` (`.rumdl.toml`), not markdownlint. Wrap prose at 80 cols;
  code blocks are exempt from the line-length rule.
- Links: `lychee` (`lychee.toml`), not markdown-link-check.
- Shell (incl. fenced code blocks in Markdown): `shellcheck` (SC2317 excluded)
  and `shfmt` (`--case-indent --indent 2 --space-redirects`). Use uppercase
  bash variables.
- `CHANGELOG.md` is excluded from all linters.
- Several config files use `keep-sorted` marker comments; keep listed entries
  sorted or the check fails.

## Commits, branches, PRs

- Conventional Commits, validated by `commit-check`. Subject lower-case
  (`CCHK_SUBJECT_CAPITALIZED: false`), max 72 chars; no trailing period.
- Branches must follow
  [Conventional Branch](https://conventional-branch.github.io/):
  `feat/`, `fix/`, `hotfix/`, `release/`, `chore/` + lowercase hyphenated name.
- PR titles must be conventional-commit format (`semantic-pull-request`).
  Open PRs as drafts. `@ruzickap` owns all paths (CODEOWNERS).

## Releases

Automated via release-please (`simple` type) on merge to `main`: it maintains
the version + `CHANGELOG.md` and, on release, `run.sh` builds the document and
uploads the assets to the release. No manual version bumping or tagging.
